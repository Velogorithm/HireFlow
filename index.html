03 — How the deep sync works

When an offer is signed, the worker arrives complete.

STEP 1
Offer signed in HireFlow
Candidate e-signs the offer letter via DocuSeal / HelloSign. Status auto-advances to Hired.
STEP 2
API call to ShiftFlow
HireFlow POSTs the hire's record (name, certs, start date, leave balance) to ShiftFlow's API.
STEP 3
Worker created in ShiftFlow
Same employee_id used in both products. First-week shifts auto-scheduled based on role default.
STEP 4
Unified audit trail
Every event in either product writes to the same audit log. Full lifecycle: applied → hired → first shift → 90-day decision.
SHARED INFRASTRUCTURE

Both products run on Cloudflare D1 in WNAM region (Standard tier) or the customer's Canadian-resident instance (Enterprise). Tenant isolation is enforced at every database query — one customer's data never crosses to another. The deep sync is technically a series of internal API calls within the same tenant_id, not a cross-organization data flow.
